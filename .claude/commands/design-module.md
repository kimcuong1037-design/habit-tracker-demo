description: "Design a module end-to-end: user story → interaction flow → wireframe → data model → API"
---

You are orchestrating the design-to-implementation pipeline for a module of the Habit Tracker app.

The user will specify which module/feature to design. Follow this pipeline:

## Step 1: User Story
- Write user stories in standard format: "As a [user], I want to [action], so that [benefit]"
- Include acceptance criteria for each story
- Save/append to `docs/product/user-stories.md`

## Step 2: Interaction Flow
- Design the step-by-step user interaction flow for this feature
- Include entry points, user actions, system responses, error states
- Save/append to `docs/design/user-flows.md`

## Step 3: Wireframe (Text-based)
- Create ASCII/text-based wireframe layouts for key screens
- Annotate interactive elements and state changes
- Save/append to `docs/design/wireframes.md`

## Step 4: Data Model Impact
- Identify what data models are needed or changed
- Save/append to `docs/architecture/database-design.md`

## Step 5: API Endpoints
- Define the REST API endpoints needed
- Include request/response schemas
- Save/append to `docs/architecture/api-design.md`

Use multiple agents in parallel where steps are independent. After completing all steps, update the progress log.

Reference PRD at `docs/product/PRD.md` for requirements. Use Chinese for discussion with the user.
