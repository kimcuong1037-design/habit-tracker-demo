# Habit Tracker — 演示指南

## 环境准备（一次性）

```bash
# 1. 安装依赖
pnpm install

# 2. 初始化数据库
pnpm db:migrate
pnpm db:seed        # 创建 demo 账号: demo / demo1234
```

## 启动演示

```bash
# 构建 + 启动生产服务器
pnpm build
pnpm start
```

浏览器打开 **http://localhost:3000**

> 开发模式也可以用 `pnpm dev`（前后端热更新），前端在 5173 端口。

---

## 演示流程（约 8-10 分钟）

### 第一幕：落地页（1 分钟）

1. 打开 http://localhost:3000，展示落地页
2. 指出 4 张**科学知识卡片**（Habit Loop、66 天法则、习惯叠加、环境设计）
3. 说明：整个产品设计基于行为心理学研究

### 第二幕：注册 & 登录（1 分钟）

**方式 A — 使用预置账号（快速）：**
- 点击「登录」，输入 `demo` / `demo1234`

**方式 B — 现场注册（展示完整流程）：**
- 点击「开始注册」，创建新账号（如 `reviewer` / `pass123456`）
- 展示表单验证（用户名 3-20 字符、密码至少 6 位）

### 第三幕：创建习惯（2 分钟）

1. 进入首页，点击「添加习惯」
2. 创建第一个习惯，例如：
   - 名称：`每天冥想 10 分钟`
   - 触发场景（Cue）：选择「触发提示」→ `早餐后`
   - 开始日期：今天
3. 创建第二个习惯，展示**习惯叠加**：
   - 名称：`喝一杯水`
   - 触发场景：选择「习惯叠加」→ 关联上一个习惯
4. 提及：**最多 5 个活跃习惯**（基于科学研究，避免分散注意力）

### 第四幕：今日打卡（2 分钟）

1. 在首页点击习惯旁的打卡按钮 ✓
2. 展示：
   - **进度条**实时更新（1/2 → 2/2）
   - **连续天数 Streak** 从 0 变为 1
   - **AI 每日鼓励语**出现（底部卡片）
3. 说明 Streak 是实时计算的（DD-003），不存储

### 第五幕：周视图（1 分钟）

1. 点击底部导航切换到「周视图」
2. 展示 7 天打卡矩阵
3. 说明可以查看每个习惯的一周打卡情况

### 第六幕：编辑 & 提醒（1 分钟）

1. 点击习惯卡片进入详情
2. 展示编辑功能（修改名称、描述）
3. 设置**提醒时间**（如 08:00）
4. 授权浏览器通知权限，说明使用 Web Notification API（DD-014）

### 第七幕：补卡机制（可选，1 分钟）

1. 如果昨天没打卡，展示「补卡」按钮
2. 说明：每月限 3 次补卡（全局配额，DD-007）
3. 体现产品的**人性化设计**——允许偶尔遗忘

## 技术亮点（可在 Q&A 环节展开）

| 亮点 | 说明 |
|------|------|
| **Monorepo 架构** | shared/client/server 三包共享类型和校验（DD-001） |
| **科学驱动设计** | 习惯环路、66 天法则、习惯叠加 → 融入产品功能 |
| **JWT 认证** | bcrypt 加密 + JWT token + 路由守卫（DD-015） |
| **Zod 全栈校验** | 前后端共用同一套 schema，类型安全 |
| **AI 鼓励** | Claude Haiku 生成个性化鼓励语，有 fallback（DD-013） |
| **里程碑系统** | 7/21/30/66/100 天自动检测 + 数据快照（DD-005） |
| **Streak 断裂安慰** | 检测连续打卡中断，显示安慰消息（DD-004） |
| **测试覆盖** | 80 个测试（Vitest + supertest），覆盖 schemas/services/routes |

## 故障排查

| 问题 | 解决 |
|------|------|
| 端口 3000 被占用 | `PORT=3001 pnpm start` |
| 数据库不存在 | `pnpm db:migrate && pnpm db:seed` |
| AI 鼓励显示固定文本 | 正常行为（未设 ANTHROPIC_API_KEY 时使用 fallback） |
| 通知不弹出 | 检查浏览器是否授权了通知权限 |
