## Epic 1: 创建习惯 (Create Habit) `P0`

### US-1.1 基础创建
**As a** 用户，**I want to** 创建一个新的每日习惯，**so that** 我可以开始追踪它的执行情况。

**验收标准：**
- [ ] 可以输入习惯名称（必填，1-50字符）
- [ ] 可以输入习惯描述（可选，0-200字符）
- [ ] 可以选择开始日期（默认今天，不可选过去的日期）
- [ ] 频次固定为"每日"，界面上展示但不可修改
- [ ] 创建成功后返回习惯列表，新习惯出现在列表中

### US-1.2 设置触发提示 (Trigger Cue)
**As a** 用户，**I want to** 在创建习惯时设置一个触发场景，**so that** 我能建立"如果-那么"的行为链接，帮助习惯养成。

**验收标准：**
- [ ] 提供预设场景选项：起床后、刷牙后、早餐后、午餐后、晚餐后、到家后、睡觉前
- [ ] 支持用户自定义输入触发场景
- [ ] 选择后生成"如果[场景]，那么[习惯]"的预览文案
- [ ] 触发提示和习惯叠加至少必须选择一种（不可全部跳过）

### US-1.3 设置习惯叠加 (Habit Stacking)
**As a** 用户，**I want to** 将新习惯绑定在一个已有习惯之后，**so that** 新习惯可以借助已有习惯的惯性更快养成。

**验收标准：**
- [ ] 展示用户已创建的活跃习惯列表供选择
- [ ] 选择后生成"在[已有习惯]之后，我将[新习惯]"的预览文案
- [ ] 当用户没有已创建的习惯时，此选项置灰并提示"创建第一个习惯后即可使用习惯叠加"
- [ ] 与触发提示互斥选择或可同时设置，但至少选其一

### US-1.4 习惯数量限制
**As a** 用户，**I want to** 在达到习惯上限时收到科学依据的提示，**so that** 我能专注于当前习惯的养成成功率。

**验收标准：**
- [ ] 活跃习惯上限为 5 个
- [ ] 达到上限时，创建按钮变为不可用状态
- [ ] 展示提示文案：「研究表明，同时培养过多习惯会分散意志力，降低成功率。建议先专注养成现有习惯，再添加新目标。」
- [ ] 用户删除或归档习惯后，恢复创建能力

## Epic 2: 查看习惯列表 (View Habit List) `P0`

### US-2.1 今日待办视图
**As a** 用户，**I want to** 打开应用时看到今日待完成的习惯列表，**so that** 我能快速了解今天还需要完成什么。

**验收标准：**
- [ ] 首屏主体区域展示未完成的习惯列表（以待办视角排列）
- [ ] 每个习惯卡片展示：习惯名称、Cue 标签（简短）、连续打卡天数（🔥 streak）、打卡复选框
- [ ] 已完成的习惯移动到列表底部"已完成"区域，视觉弱化
- [ ] 列表为空时展示空状态引导（"还没有习惯？创建你的第一个吧"）

### US-2.2 今日进度概览
**As a** 用户，**I want to** 在页面顶部看到今日整体进度，**so that** 我一眼就知道今天的完成情况。

**验收标准：**
- [ ] 顶部常驻轻量进度条，格式：`▓▓▓░░ 3/5 已完成`
- [ ] 进度随打卡实时更新
- [ ] 全部完成时进度条变为醒目完成色，显示庆祝文案
- [ ] 无习惯时不展示进度条

## Epic 3: 今日打卡 (Daily Check-in) `P0`

### US-3.1 一步打卡
**As a** 用户，**I want to** 点击一下就完成某个习惯的打卡，**so that** 日常记录尽可能简单快速。

**验收标准：**
- [ ] 点击习惯卡片上的复选框即完成打卡
- [ ] 打卡后该习惯从"待完成"区平滑移动到"已完成"区
- [ ] 打卡操作响应时间 < 200ms
- [ ] 今日已打卡的习惯复选框为已选状态，不可重复打卡

### US-3.2 日常打卡反馈
**As a** 用户，**I want to** 在完成打卡后收到轻松的鼓励反馈，**so that** 我感受到正向的奖赏激励。

**验收标准：**
- [ ] 打卡后显示轻量 toast 提示，风格为朋友式（如「不错嘛，又搞定一个！」）
- [ ] 伴随轻微的打勾动画效果
- [ ] toast 在 2-3 秒后自动消失
- [ ] 鼓励文案从预设文案库中随机选取，避免重复感

### US-3.3 全部完成庆祝
**As a** 用户，**I want to** 在今日所有习惯全部完成时看到特别的庆祝反馈，**so that** 我获得更强的成就感。

**验收标准：**
- [ ] 最后一个习惯打卡后触发特殊庆祝效果
- [ ] 进度条填满并变色
- [ ] 显示温暖文案（如「今天全部达成！你对自己的承诺从未落空。」）
- [ ] 庆祝效果包含适当的视觉动效（灵动但不过分）

## Epic 4: 编辑习惯 (Edit Habit) `P0`

### US-4.1 编辑基础信息
**As a** 用户，**I want to** 修改习惯的名称、描述和开始日期，**so that** 我可以调整习惯的定义。

**验收标准：**
- [ ] 从习惯卡片进入编辑页面，表单预填当前值
- [ ] 可编辑字段：名称、描述、开始日期
- [ ] 保存后返回列表，信息立即更新
- [ ] 编辑不影响已有打卡记录和 Streak

### US-4.2 编辑 Cue 设置
**As a** 用户，**I want to** 修改习惯的触发提示或叠加绑定，**so that** 我能找到更适合自己的触发方式。

**验收标准：**
- [ ] 可在触发提示和习惯叠加之间切换
- [ ] 修改 Cue 时显示引导提示：「调整触发场景有助于重新找到适合你的节奏」
- [ ] 仍然需要至少选择一种 Cue 方式
- [ ] Cue 修改不影响 Streak

## Epic 5: 删除习惯 (Delete Habit) `P0`

### US-5.1 确认删除
**As a** 用户，**I want to** 删除不再需要的习惯，**so that** 我可以保持列表整洁，腾出名额给新习惯。

**验收标准：**
- [ ] 从习惯卡片操作菜单触发删除
- [ ] 弹出确认对话框，文案：「确定要删除"[习惯名称]"吗？所有打卡记录将一并删除，且无法恢复。」
- [ ] 提供"取消"和"确认删除"两个按钮，"确认删除"为危险色
- [ ] 确认后习惯从列表消失，进度条更新
- [ ] 删除后活跃习惯数 -1，释放创建名额

## Epic 7: 连续打卡 / Streak `P0`

### US-7.1 Streak 计算与展示
**As a** 用户，**I want to** 看到每个习惯的连续打卡天数，**so that** 我有动力保持连续。

**验收标准：**
- [ ] 每个习惯卡片上展示 🔥 + 连续天数
- [ ] 计算规则：今日打卡 + 昨日打卡 → streak += 1；今日打卡 + 昨日未打卡 → streak = 1
- [ ] Streak 为 0 时不显示火焰图标
- [ ] 新创建的习惯在首次打卡后 streak = 1

## Epic 8: 补卡机制 (Retroactive Check-in) `P0`

### US-8.1 补卡提示
**As a** 用户，**I want to** 在遗漏昨日打卡时看到补卡提示，**so that** 我可以挽救即将断裂的 streak。

**验收标准：**
- [ ] 当某习惯昨日未打卡且今日尚未打卡时，卡片上显示"昨日未打卡，补卡？"标签
- [ ] 仅当该习惯的 streak > 0（即昨天之前有连续记录）或开始日期为昨天时才显示
- [ ] 补卡提示样式与常规打卡区分（如使用不同颜色标签）

### US-8.2 执行补卡
**As a** 用户，**I want to** 补打昨天遗漏的卡，**so that** 我的连续记录不会因一天的疏忽而断裂。

**验收标准：**
- [ ] 点击补卡提示后弹出确认：「为[习惯名称]补打昨日（[日期]）的卡？」
- [ ] 确认后昨日记录为已打卡，标记为补卡（isRetroactive = true）
- [ ] 补卡后 streak 正常延续
- [ ] 仅可补打昨天的卡，不可补更早的日期

### US-8.3 补卡配额管理
**As a** 用户，**I want to** 了解本月剩余的补卡次数，**so that** 我能合理使用补卡机会。

**验收标准：**
- [ ] 每月 3 次补卡配额，自然月重置
- [ ] 补卡确认弹窗中显示"本月剩余补卡：X/3"
- [ ] 配额用尽时补卡入口置灰，提示「本月补卡次数已用完，下月将重置」
- [ ] 配额在每月 1 日 00:00 自动重置

## Epic 9: 里程碑回顾 (Milestone Review) `P0`

### US-9.1 里程碑触发
**As a** 用户，**I want to** 在达到关键打卡天数时收到阶段性回顾，**so that** 我能看到自己的进步并获得深度激励。

**验收标准：**
- [ ] 在 7 天、21 天、30 天、66 天、100 天时触发里程碑回顾
- [ ] 以当次打卡完成时弹出回顾卡片的形式展示
- [ ] 每个里程碑每个习惯只触发一次

### US-9.2 回顾内容展示
**As a** 用户，**I want to** 在里程碑回顾中看到数据统计和科学鼓励，**so that** 我理解自己行为改变的意义。

**验收标准：**
- [ ] 数据区域展示：总打卡天数、完成率、最长连续天数
- [ ] 鼓励文案区域：根据里程碑等级展示有科学深度的文字
  - 7天：导师式「连续 7 天！你的大脑正在建立新的神经通路，坚持下去。」
  - 21天：深度「21 天！许多人相信这是习惯形成的起点。真正的变化，从神经层面已经开始。」
  - 66天：诗意「66 天——研究表明，这是一个习惯从'努力坚持'变为'自然而然'的平均时间。你做到了。」
- [ ] 提供"知道了"按钮关闭回顾卡片
- [ ] 关闭后不再重复弹出同一里程碑

## Epic 10: Streak 断裂处理 `P0`

### US-10.1 断裂安慰
**As a** 用户，**I want to** 在连续打卡中断后收到温暖的安慰反馈，**so that** 我不会因为一次中断而放弃。

**验收标准：**
- [ ] 当用户打开应用，某习惯 streak 已归零时，展示安慰文案
- [ ] 文案内容：「没关系。研究表明，偶尔的中断并不会前功尽弃——你之前建立的神经通路仍然在那里。重新开始，比第一次容易得多。」
- [ ] 同一次断裂只展示一次安慰消息
- [ ] 安慰消息与补卡提示可同时出现（先安慰，引导补卡）

## Epic 11: 周视图 (Week View) `P1`

### US-11.1 本周打卡矩阵
**As a** 用户，**I want to** 查看本周 7 天的打卡情况，**so that** 我能快速回顾本周表现。

**验收标准：**
- [ ] 展示 7 列（周一至周日）矩阵视图
- [ ] 每行对应一个习惯
- [ ] 已打卡 = 填充色块，未打卡 = 空心/浅色
- [ ] 今日列高亮标识
- [ ] 可切换至上一周/下一周

## Epic 12: 主题切换 (Theme Switch) `P2`

### US-12.1 深色/浅色模式
**As a** 用户，**I want to** 在深色和浅色模式间切换，**so that** 我能在不同环境下舒适地使用应用。

**验收标准：**
- [ ] 支持浅色、深色两种模式
- [ ] 提供手动切换入口（设置页或顶部按钮）
- [ ] 支持"跟随系统"选项
- [ ] 用户偏好持久化存储

---

## Epic 13: 月视图 (Month View) `P3`

### US-13.1 月度热力图
**As a** 用户，**I want to** 查看当月的打卡热力图，**so that** 我能一览整月的习惯执行密度。

**验收标准：**
- [ ] 类似 GitHub Contribution 热力图样式
- [ ] 颜色深浅表示当日完成度（如完成 0/5 到 5/5 对应不同深度）
- [ ] 可切换到上一月/下一月
- [ ] 展示月度统计：总完成率、活跃天数
