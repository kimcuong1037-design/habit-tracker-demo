# Habit Tracker - 产品需求文档 (PRD)

> 版本：v1.0 | 更新日期：2026-03-10 | 状态：Draft

---

## 1. 产品定位

### 1.1 一句话描述
一款融合行为科学原理的习惯打卡工具，帮助用户以科学的方式建立并巩固每日习惯。

### 1.2 核心理念
**"少即是多，科学养成"**
- 日常操作极简高效，不给用户增加认知负担
- 在关键节点（启动、里程碑、受阻）提供温暖且有科学依据的引导
- 深度融入习惯科学（习惯回路、习惯叠加、66天法则），但通过默认选项降低上手门槛

### 1.3 目标用户
兼顾新手与有基础的用户：
- **新手：** 在创建习惯和关键里程碑时获得充分引导
- **有基础用户：** 日常使用简洁高效，不被过度打扰

---

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| 科学驱动 | 所有功能设计基于"习惯回路"（提示→惯例→奖赏）理论 |
| 简洁日常 | 每日打卡操作 ≤ 2 步完成，界面干净无干扰 |
| 温暖关怀 | 关键时刻提供拟人化、有智慧的文字反馈 |
| 专注聚焦 | 鼓励用户专注少量习惯（≤5个），提高养成成功率 |
| 可扩展 | 架构预留用户认证扩展能力 |

## 3. 功能规格

### 3.1 创建习惯（功能 1）

**基础字段：**
| 字段 | 必填 | 说明 |
|------|------|------|
| 习惯名称 | 是 | 如"阅读 20 分钟" |
| 习惯描述 | 否 | 可选的补充说明 |
| 开始日期 | 是 | 默认今天 |
| 频次 | 是 | 固定为"每日"（v1） |

**科学增强字段（必选其一）：**

用户在创建习惯时，**必须**至少选择以下一种 Cue 设置方式：

**方式 A - 触发提示 (Trigger Cue)：**
- 引导问题：「你打算在什么场景下执行这个习惯？」
- 提供常见默认选项：
  - 🌅 起床后
  - 🪥 刷牙后
  - 🍽️ 早餐后 / 午餐后 / 晚餐后
  - 🏠 到家后
  - 🌙 睡觉前
- 支持用户自定义输入

**方式 B - 习惯叠加 (Habit Stacking)：**
- 引导问题：「你想把这个习惯绑定在哪个已有习惯之后？」
- 展示用户已创建的习惯列表供选择
- 选择后生成文案：「在 [已有习惯] 之后，我将 [新习惯]」

**习惯数量限制：**
- 活跃习惯上限：**5 个**
- 达到上限时提示：「研究表明，同时培养过多习惯会分散意志力，降低成功率。建议先专注养成现有习惯，再添加新目标。」

### 3.2 习惯列表（功能 2）

**主界面布局（首屏）：**

```
┌─────────────────────────────────┐
│  今日进度 ▓▓▓▓▓░░░  3/5        │  ← 轻量进度条（顶部常驻）
├─────────────────────────────────┤
│                                 │
│  ☐ 阅读 20 分钟                │  ← 待完成（优先展示）
│    🔗 早餐后 · 🔥 12 天        │
│                                 │
│  ☐ 运动 30 分钟                │
│    🔗 到家后 · 🔥 3 天         │
│                                 │
│  ─ ─ ─ 已完成 ─ ─ ─           │
│                                 │
│  ☑ 喝水 8 杯          🔥 15 天 │  ← 已完成（折叠/置底）
│  ☑ 背单词             🔥 7 天  │
│  ☑ 早睡               🔥 21 天 │
│                                 │
└─────────────────────────────────┘
```

**信息优先级：**
1. 顶部：轻量化今日进度概览（进度条 + 数字）
2. 主体：待完成习惯列表（以待办视角驱动行动）
3. 底部：已完成习惯（视觉弱化但可查看）

**每个习惯卡片展示：**
- 习惯名称
- 今日完成状态（复选框）
- 关联的 Cue 提示（简短标签）
- 连续打卡天数（🔥 streak）
- 操作入口（编辑/删除）

### 3.3 今日打卡（功能 3）

**操作方式：**
- 在列表中直接点击复选框完成打卡（一步操作）
- 今日已打卡的习惯：复选框变为已选状态，不可重复打卡
- 打卡后该习惯从"待完成"区移动到"已完成"区

**打卡反馈 -- 分层奖赏系统：**

| 场景 | 反馈类型 | 示例 |
|------|----------|------|
| 日常打卡 | 轻松朋友式 + 轻微视觉动效 | 「不错嘛，又搞定一个！」+ 打勾动画 |
| 今日全部完成 | 温暖鼓励 + 进度条填满动效 | 「今天全部达成！你对自己的承诺从未落空。」|
| Streak 里程碑（7天） | 导师式科学鼓励 | 「连续 7 天！你的大脑正在建立新的神经通路，坚持下去。」|
| Streak 里程碑（21天） | 深度激励 | 「21 天！许多人相信这是习惯形成的起点。真正的变化，从神经层面已经开始。」|
| Streak 里程碑（66天） | 诗意庆祝 | 「66 天——研究表明，这是一个习惯从'努力坚持'变为'自然而然'的平均时间。你做到了。」|

### 3.4 编辑习惯（功能 4）

- 可编辑字段：名称、描述、开始日期、Cue 设置
- 编辑后保留原有打卡记录和 Streak 数据
- 修改 Cue 时展示引导提示：「调整触发场景有助于重新找到适合你的节奏」

### 3.5 删除习惯（功能 5）

- 点击删除后弹出确认对话框
- 确认文案：「确定要删除"[习惯名称]"吗？所有打卡记录将一并删除，且无法恢复。」
- 需要用户明确点击"确认删除"按钮

### 3.6 今日进度（功能 6）

- 顶部常驻轻量进度条
- 格式：`▓▓▓░░ 3/5 已完成`
- 全部完成时：进度条变为醒目的完成色 + 庆祝文案

### 3.7 连续打卡天数 / Streak（功能 7）

**计算规则：**
- 今日打卡 + 昨日已打卡 → streak += 1
- 今日打卡 + 昨日未打卡 → streak = 1
- 今日未打卡 → streak 不变（等待打卡或次日重置）

**补卡机制：**
- 允许补打**昨天**的卡（仅限昨天，不可补更早的）
- 每月限定 **3 次** 补卡机会
- 补卡后 streak 正常延续
- 补卡入口：习惯卡片上显示"昨日未打卡，补卡？"提示（仅在昨日遗漏时显示）

**Streak 断裂安慰：**
- 当用户打开应用且某习惯 streak 已归零时显示：
- 「没关系。研究表明，偶尔的中断并不会前功尽弃——你之前建立的神经通路仍然在那里。重新开始，比第一次容易得多。」
- 显示一次后不再重复（同一次断裂）

## 4. 阶段性回顾系统（里程碑触发）

### 4.1 触发节点

| 天数 | 意义 | 回顾内容 |
|------|------|----------|
| 7 天 | 第一周完成 | 基础数据 + 鼓励继续 |
| 21 天 | 传统习惯节点 | 数据回顾 + 科学解读 |
| 30 天 | 满月 | 月度完整报告 |
| 66 天 | 科学习惯形成点 | 深度庆祝 + 全面数据 |
| 100 天 | 百日里程碑 | 成就庆祝 |

### 4.2 回顾内容

**数据展示：**
- 总打卡天数 / 应打卡天数
- 完成率百分比
- 最长连续天数
- 本阶段 streak 变化趋势（简单折线或柱状）

**文字激励：**
- 结合科学文档内容，为每个里程碑撰写有深度的鼓励文案
- 融入科学知识点（如多巴胺强化、神经通路建立等）

## 5. 选做功能（按优先级排序）

### 5.1 周视图（P1 - 优先实现）
- 展示本周 7 天的打卡矩阵
- 每个习惯一行，7 列表示周一到周日
- 已打卡 = 填充色块，未打卡 = 空心
- 便于用户快速回顾本周表现

### 5.2 主题切换（P2）
- 支持浅色 / 深色模式
- 跟随系统设置 or 手动切换
- 存储用户偏好

### 5.3 月视图（P3）
- 类似 GitHub Contribution 热力图
- 按日展示当月打卡密度
- 颜色深浅表示完成度

### 5.4 分类标签（暂不实现）
- v1 不做，后续版本考虑

## 6. 技术方案概要

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | TypeScript + React + Vite | 现代前端工具链，开发体验好 |
| UI 框架 | Tailwind CSS | 快速构建简洁界面，高度可定制 |
| 后端 | Node.js + Express + TypeScript | 轻量灵活，与前端共享类型 |
| 数据库 | SQLite (via Prisma ORM) | 零配置，本地开发友好，后续可迁移 PostgreSQL |
| ORM | Prisma | 类型安全，schema-first，迁移方便 |
| API 风格 | RESTful | 简单直接，适合 CRUD 为主的场景 |
| 认证预留 | 中间件 + User 表预留 | v1 单用户模式，架构支持后续接入 JWT/OAuth |

## 7. 核心数据模型（初步）

```
Habit
├── id: string (UUID)
├── name: string
├── description: string?
├── startDate: Date
├── frequency: "daily"
├── cueType: "trigger" | "stacking"
├── cueValue: string          # 触发提示文本 或 关联习惯ID
├── isActive: boolean
├── createdAt: DateTime
└── updatedAt: DateTime

CheckIn
├── id: string (UUID)
├── habitId: string (FK → Habit)
├── date: Date                 # 打卡日期（仅日期，无时间）
├── isRetroactive: boolean     # 是否为补卡
├── createdAt: DateTime
└── updatedAt: DateTime

MilestoneEvent
├── id: string (UUID)
├── habitId: string (FK → Habit)
├── type: "7d" | "21d" | "30d" | "66d" | "100d"
├── triggeredAt: DateTime
├── dismissed: boolean         # 用户是否已查看
└── createdAt: DateTime

User (预留)
├── id: string (UUID)
├── email: string?
├── createdAt: DateTime
└── updatedAt: DateTime
```

## 8. 非功能需求

| 项目 | 要求 |
|------|------|
| 响应式 | 移动端优先，兼顾桌面端 |
| 性能 | 首屏加载 < 2s，打卡操作 < 200ms 反馈 |
| 数据安全 | 本地 SQLite，无敏感数据外传 |
| 可扩展 | 架构支持后续接入用户认证系统 |

## 附录：科学原理与功能映射

| 科学原理 | 对应功能 | 融入方式 |
|----------|----------|----------|
| 习惯回路（提示→惯例→奖赏） | 创建习惯时设置 Cue + 打卡后奖赏反馈 | 必选 Cue + 分层反馈 |
| 习惯叠加 | 创建习惯时绑定已有习惯 | Habit Stacking 选项 |
| 66天法则 | 里程碑回顾系统 | 66天触发深度庆祝 |
| 意志力有限 | 习惯数量限制（≤5） | 硬性上限 + 科学解释 |
| 偶尔中断不致命 | Streak 断裂安慰 + 补卡机制 | 温暖文案 + 月3次补卡 |
| 实施意图 | Trigger Cue 设置 | "如果[场景]，那么[习惯]" |
