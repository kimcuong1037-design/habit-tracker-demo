# Habit Tracker - 产品需求文档 (PRD)

> 版本：v1.0 | 更新日期：2026-03-10 | 状态：Draft

---

## 1. 产品定位

### 1.1 一句话描述
一款融合行为科学原理的习惯打卡工具，帮助用户以科学的方式建立并巩固每日习惯。

### 1.2 核心理念
**"少即是多，科学养成"**
- 日常操作极简高效，不给用户增加认知负担
- 在关键节点（启动、里程碑、受阻）提供温暖且有科学依据的引导
- 深度融入习惯科学（习惯回路、习惯叠加、66天法则），但通过默认选项降低上手门槛

### 1.3 目标用户
兼顾新手与有基础的用户：
- **新手：** 在创建习惯和关键里程碑时获得充分引导
- **有基础用户：** 日常使用简洁高效，不被过度打扰

---

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| 科学驱动 | 所有功能设计基于"习惯回路"（提示→惯例→奖赏）理论 |
| 简洁日常 | 每日打卡操作 ≤ 2 步完成，界面干净无干扰 |
| 温暖关怀 | 关键时刻提供拟人化、有智慧的文字反馈 |
| 专注聚焦 | 鼓励用户专注少量习惯（≤5个），提高养成成功率 |
| 可扩展 | 架构预留用户认证扩展能力 |

## 3. 功能规格

### 3.1 创建习惯（功能 1）

**基础字段：**
| 字段 | 必填 | 说明 |
|------|------|------|
| 习惯名称 | 是 | 如"阅读 20 分钟" |
| 习惯描述 | 否 | 可选的补充说明 |
| 开始日期 | 是 | 默认今天 |
| 频次 | 是 | 固定为"每日"（v1） |

**科学增强字段（必选其一）：**

用户在创建习惯时，**必须**至少选择以下一种 Cue 设置方式：

**方式 A - 触发提示 (Trigger Cue)：**
- 引导问题：「你打算在什么场景下执行这个习惯？」
- 提供常见默认选项：
  - 🌅 起床后
  - 🪥 刷牙后
  - 🍽️ 早餐后 / 午餐后 / 晚餐后
  - 🏠 到家后
  - 🌙 睡觉前
- 支持用户自定义输入

**方式 B - 习惯叠加 (Habit Stacking)：**
- 引导问题：「你想把这个习惯绑定在哪个已有习惯之后？」
- 展示用户已创建的习惯列表供选择
- 选择后生成文案：「在 [已有习惯] 之后，我将 [新习惯]」

**习惯数量限制：**
- 活跃习惯上限：**5 个**
- 达到上限时提示：「研究表明，同时培养过多习惯会分散意志力，降低成功率。建议先专注养成现有习惯，再添加新目标。」

### 3.2 习惯列表（功能 2）

**主界面布局（首屏）：**

```
┌─────────────────────────────────┐
│  今日进度 ▓▓▓▓▓░░░  3/5        │  ← 轻量进度条（顶部常驻）
├─────────────────────────────────┤
│                                 │
│  ☐ 阅读 20 分钟                │  ← 待完成（优先展示）
│    🔗 早餐后 · 🔥 12 天        │
│                                 │
│  ☐ 运动 30 分钟                │
│    🔗 到家后 · 🔥 3 天         │
│                                 │
│  ─ ─ ─ 已完成 ─ ─ ─           │
│                                 │
│  ☑ 喝水 8 杯          🔥 15 天 │  ← 已完成（折叠/置底）
│  ☑ 背单词             🔥 7 天  │
│  ☑ 早睡               🔥 21 天 │
│                                 │
└─────────────────────────────────┘
```

**信息优先级：**
1. 顶部：轻量化今日进度概览（进度条 + 数字）
2. 主体：待完成习惯列表（以待办视角驱动行动）
3. 底部：已完成习惯（视觉弱化但可查看）

**每个习惯卡片展示：**
- 习惯名称
- 今日完成状态（复选框）
- 关联的 Cue 提示（简短标签）
- 连续打卡天数（🔥 streak）
- 操作入口（编辑/删除）

### 3.3 今日打卡（功能 3）

**操作方式：**
- 在列表中直接点击复选框完成打卡（一步操作）
- 今日已打卡的习惯：复选框变为已选状态，不可重复打卡
- 打卡后该习惯从"待完成"区移动到"已完成"区

**打卡反馈 -- 分层奖赏系统：**

| 场景 | 反馈类型 | 示例 |
|------|----------|------|
| 日常打卡 | 轻松朋友式 + 轻微视觉动效 | 「不错嘛，又搞定一个！」+ 打勾动画 |
| 今日全部完成 | 温暖鼓励 + 进度条填满动效 | 「今天全部达成！你对自己的承诺从未落空。」|
| Streak 里程碑（7天） | 导师式科学鼓励 | 「连续 7 天！你的大脑正在建立新的神经通路，坚持下去。」|
| Streak 里程碑（21天） | 深度激励 | 「21 天！许多人相信这是习惯形成的起点。真正的变化，从神经层面已经开始。」|
| Streak 里程碑（66天） | 诗意庆祝 | 「66 天——研究表明，这是一个习惯从'努力坚持'变为'自然而然'的平均时间。你做到了。」|

### 3.4 编辑习惯（功能 4）

- 可编辑字段：名称、描述、开始日期、Cue 设置
- 编辑后保留原有打卡记录和 Streak 数据
- 修改 Cue 时展示引导提示：「调整触发场景有助于重新找到适合你的节奏」

### 3.5 删除习惯（功能 5）

- 点击删除后弹出确认对话框
- 确认文案：「确定要删除"[习惯名称]"吗？所有打卡记录将一并删除，且无法恢复。」
- 需要用户明确点击"确认删除"按钮

### 3.6 今日进度（功能 6）

- 顶部常驻轻量进度条
- 格式：`▓▓▓░░ 3/5 已完成`
- 全部完成时：进度条变为醒目的完成色 + 庆祝文案

### 3.7 连续打卡天数 / Streak（功能 7）

**计算规则：**
- 今日打卡 + 昨日已打卡 → streak += 1
- 今日打卡 + 昨日未打卡 → streak = 1
- 今日未打卡 → streak 不变（等待打卡或次日重置）

**补卡机制：**
- 允许补打**昨天**的卡（仅限昨天，不可补更早的）
- 每月限定 **3 次** 补卡机会
- 补卡后 streak 正常延续
- 补卡入口：习惯卡片上显示"昨日未打卡，补卡？"提示（仅在昨日遗漏时显示）

**Streak 断裂安慰：**
- 当用户打开应用且某习惯 streak 已归零时显示：
- 「没关系。研究表明，偶尔的中断并不会前功尽弃——你之前建立的神经通路仍然在那里。重新开始，比第一次容易得多。」
- 显示一次后不再重复（同一次断裂）

### 3.8 AI 每日鼓励（功能 8）

**功能定位：**
每天为用户生成一句个性化的鼓励话语，结合当前习惯状态，附带表情包/颜文字，让应用更有温度和灵动感。

**触发时机：**
- 用户打开首页时，在习惯列表上方展示当日鼓励语
- 每个习惯每天生成一次，当日内不重复生成

**生成依据（Context）：**
| 状态信号 | 说明 |
|----------|------|
| 当前 streak 天数 | 刚起步 / 稳定坚持 / 接近里程碑 |
| 今日打卡进度 | 未开始 / 部分完成 / 全部完成 |
| 是否刚经历 streak 断裂 | 切换为安慰模式 |
| 习惯名称 | 让鼓励语更具针对性 |

**输出风格：**
- 语气：温暖朋友式，像一个懂科学的好友在打气
- 长度：1-2 句话，简短有力
- 必须包含至少一个颜文字或表情符号，增加灵动感
- 风格随状态自然切换（日常→轻松、断裂→温暖安慰、接近里程碑→兴奋冲刺）

**示例：**

| 场景 | 鼓励语示例 |
|------|-----------|
| 连续 5 天 | 「五天了！你的节奏越来越稳了 ᕙ(⇀‸↼‶)ᕗ」 |
| 刚断裂 | 「没关系，休息也是旅程的一部分 (´；ω；\`)♡ 明天继续就好」 |
| 接近 21 天里程碑 | 「还差 3 天就 21 天里程碑了！冲鸭 ٩(ˊᗜˋ*)و」 |
| 今日全部完成 | 「全部搞定！今天的你超棒的 ✧(≖ ◡ ≖✿)」 |
| 刚开始第 1 天 | 「万事开头难，但你已经迈出第一步了 (ﾉ◕ヮ◕)ﾉ*:・ﾟ✧」 |

**技术方案：**

| 方面 | 方案 |
|------|------|
| AI 模型 | Claude Haiku（成本低、速度快，生成短文案绰绰有余） |
| 调用方式 | 服务端通过 `@anthropic-ai/sdk` 单次 API 调用（见 DD-013） |
| 调用时机 | `GET /api/today` 聚合接口中生成并返回 |
| 缓存策略 | 每个习惯每天生成一次，缓存至当日结束，避免重复调用 |
| Fallback | 预置 20-30 条静态鼓励语（含颜文字），API 不可用时随机选取 |
| 成本 | 每天最多 5 习惯 × ~100 tokens = ~500 tokens/天，成本可忽略 |

## 4. 阶段性回顾系统（里程碑触发）

### 4.1 触发节点

| 天数 | 意义 | 回顾内容 |
|------|------|----------|
| 7 天 | 第一周完成 | 基础数据 + 鼓励继续 |
| 21 天 | 传统习惯节点 | 数据回顾 + 科学解读 |
| 30 天 | 满月 | 月度完整报告 |
| 66 天 | 科学习惯形成点 | 深度庆祝 + 全面数据 |
| 100 天 | 百日里程碑 | 成就庆祝 |

### 4.2 回顾内容

**数据展示：**
- 总打卡天数 / 应打卡天数
- 完成率百分比
- 最长连续天数
- 本阶段 streak 变化趋势（简单折线或柱状）

**文字激励：**
- 结合科学文档内容，为每个里程碑撰写有深度的鼓励文案
- 融入科学知识点（如多巴胺强化、神经通路建立等）

## 9. 习惯提醒通知（功能 9 — 追加需求）

### 9.1 功能定位

为习惯养成提供轻量级的定时提醒能力，在用户设定的时间通过浏览器系统通知（Web Notification API）提醒用户完成打卡。这是对现有"Cue（触发提示）"机制的补充——Cue 提供心理层面的触发锚点，而提醒通知提供时间层面的物理触发。

### 9.2 设计原则

| 原则 | 说明 |
|------|------|
| 轻量非侵入 | 提醒为可选项，不强制设置；提醒频率适度（每习惯每天最多一次） |
| 浏览器原生 | 使用 Web Notification API，无需 Push Server 或原生 App 依赖 |
| 优雅降级 | 用户拒绝通知权限或浏览器不支持时，不影响其他功能 |

### 9.3 提醒时间设置

**入口：** 创建习惯 / 编辑习惯表单中新增可选字段。

| 字段 | 必填 | 说明 |
|------|------|------|
| 提醒时间 | 否 | 格式 `HH:mm`（如 `08:00`、`21:30`），为空表示不设置提醒 |

**交互规则：**
- 创建习惯步骤一新增"提醒时间"字段，放在"频次"字段下方
- 点击后弹出时间选择器（小时:分钟）
- 提供快捷预设：早上 8:00 / 中午 12:00 / 晚上 20:00
- 可清除已设置的提醒时间（恢复为不提醒）
- 编辑习惯时同样可修改提醒时间

### 9.4 系统通知（P0）

**触发机制：**
- 应用在前台运行时，客户端定时检查（每分钟一次），到达提醒时间时触发浏览器通知
- 应用不在前台但 Service Worker 已注册时，由 Service Worker 在后台触发通知
- 今日已打卡的习惯不再发送提醒

**通知内容：**
```
标题：⏰ 习惯打卡提醒
正文：「阅读 20 分钟」— 现在是你的习惯时间！
操作：点击通知 → 聚焦应用窗口，定位到对应习惯
```

**权限管理：**
- 首次设置提醒时间时，请求浏览器通知权限（`Notification.requestPermission()`）
- 用户拒绝权限：提示「需要通知权限才能发送提醒，请在浏览器设置中允许」
- 用户未授权时，提醒时间仍可设置和保存，但不会实际发送通知

### 9.5 页面 Badge 提示（Nice-to-have）

**功能：** 在浏览器标签页标题上显示未完成习惯数量。

**实现方式：**
- 修改 `document.title`，格式：`(3) Habit Tracker` — 表示 3 个习惯待完成
- 全部完成或无习惯时恢复为 `Habit Tracker`
- 实时更新：打卡后立即更新数字

**技术方案：**

| 方面 | 方案 |
|------|------|
| 通知 API | Web Notification API（浏览器原生，see DD-014） |
| 定时机制 | 前台：`setInterval` 每分钟检查；后台：Service Worker `setTimeout` |
| 权限 | 首次设置时请求，拒绝后不影响其他功能 |
| 存储 | `reminderTime` 字段存储在 Habit 表中（`HH:mm` 或 null） |
| Badge | `document.title` 动态修改 |
| Service Worker | 注册 `sw-reminder.js`，接收消息调度通知 |

## 附录：科学原理与功能映射

| 科学原理 | 对应功能 | 融入方式 |
|----------|----------|----------|
| 习惯回路（提示→惯例→奖赏） | 创建习惯时设置 Cue + 打卡后奖赏反馈 | 必选 Cue + 分层反馈 |
| 习惯叠加 | 创建习惯时绑定已有习惯 | Habit Stacking 选项 |
| 66天法则 | 里程碑回顾系统 | 66天触发深度庆祝 |
| 意志力有限 | 习惯数量限制（≤5） | 硬性上限 + 科学解释 |
| 偶尔中断不致命 | Streak 断裂安慰 + 补卡机制 | 温暖文案 + 月3次补卡 |
| 实施意图 | Trigger Cue 设置 | "如果[场景]，那么[习惯]" |
| 环境提示强化 | 定时提醒通知（功能 9） | Cue 提供心理锚点 + 通知提供时间锚点 |
