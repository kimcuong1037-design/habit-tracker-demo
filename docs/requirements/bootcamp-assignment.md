# 题目：Habit Tracker（习惯打卡应用）
---
## 1. 产品背景

设计并实现一个面向个人用户的习惯打卡应用，帮助用户记录每日习惯执行情况，并看到自己的完成进度与连续打卡表现。

## 2. 核心用户

个人用户，希望通过一个简单清晰的工具记录自己每天是否完成某些习惯，例如：
- 运动 30 分钟
- 阅读 20 分钟
- 早睡
- 喝水 8 杯
- 背单词

## 3. 必做功能

### 功能 1：创建习惯

用户可以新增一个习惯，至少包含以下字段：
- 习惯名称
- 习惯描述（可选）
- 开始日期
- 频次（统一叫做"每日"）

### 功能 2：查看习惯列表

用户可以看到自己创建的所有习惯，列表中至少展示：
- 习惯名称
- 今日是否完成
- 连续打卡天数
- 基本操作入口

### 功能 3：今日打卡

用户可以对某个习惯进行"今日完成"打卡。
要求至少考虑：
- 今日已打卡时不能重复打卡
- 打卡后状态能正确更新

### 功能 4：编辑习惯

用户可以编辑习惯名称、描述、开始日期等基础信息。

### 功能 5：删除习惯

用户可以删除某个习惯，并有明确的确认方式。

### 功能 6：展示今日进度

系统需要展示今日整体进度，例如：
- 今日 5 个习惯，已完成 3 个
- 可用数字、进度条或其他简单方式表达

### 功能 7：连续打卡天数

每个习惯需要展示 streak，即连续打卡天数。
为了避免范围失控，建议统一规则：
如果用户今天完成打卡，并且昨天也完成了，则连续天数 +1；如果中间断了一天，则从 1 重新开始。

## 4. 选做功能

- 主题切换
- 周视图
- 月视图
- 分类标签

### 追加需求 1：习惯提醒通知

用户可以为每个习惯设置提醒时间，到达设定时间时通过系统通知提醒用户完成打卡。

**核心要求：**
- 创建/编辑习惯时可设置提醒时间（可选）
- 到达提醒时间时发送系统通知（浏览器 Notification）
- 页面标题/图标上显示未完成任务数量（badge，nice-to-have）
